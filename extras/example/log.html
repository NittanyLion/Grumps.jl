1: read the complete manual on github
2: read the paper at http://joris.pinkse.org/paper/grumps/
3: this is a preliminary version: bugs are to be expected, sanity checking and documentation are incomplete
4: still need to put in designed quadrature, so go easy on the random coefficients for now
5: does not do CUDA
6: uses multithreading, but no distributed computing
7: because of the way the random numbers for MC integration are drawn / multithreading, successive optimizations with the same data may not produce the exact same results
8: the gmm version should not be used: stick with grumps (or the cheap version thereof), mle, and sc: gmm is inferior to begin with and the gmm code is not ready for prime time
9: if you see unexpected behavior, please send me code and data so that I can replicate
10: all of the user-level commands are documented; e.g. use ?Save
11: please check the memory conservation documentation if memory becomes an issue
12: please direct all questions/suggestions/comments to Joris Pinkse at joris@psu.edu
